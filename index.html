This course is the first time I have ever engaged with writing code on my own. I had a very general idea of the theory behind a coder writing code, and browsers interpreting different tags to show the end user a website, but I had no previous knowledge of what each tag does and how to organize it all in a program like GitHub.

At first, I was following simple instructions in class to create a header, a menu, and skeleton links to all of the assignment submissions we will have throughout the course. Looking around the classroom I noticed all of our websites looked very similar. One day in class, we were asked to write about some of the design elements we wanted to incorporate to make our websites unique. As I was searching for inspiration, I came across a blog from an Italian designer with an interesting method implemented to browse the site (http://essential.riccardoborchi.it/). I later learned that this style is called "parallax" and it is trending in the industry right now. I think this may be a challenge, considering my lack of experience in coding, but I am excited to give it a shot. If I am able to successfully implement it, I think it will make my site appear very professional, considering the majority of the content is short-form text.

When thinking about a color scheme, I immediately thought of https://www.apple.com/ because I hold their site in high regard. I think it is extremely clean-cut but comes off as trendy and advanced. For this reason, I came up with a color palette to implement in my CSS styling. I have a black header with white text, a dark gray menu with white text, a light silver background for the body with black text, and a black footer with white text to match the header. I also have two shades of blue that I think complement the black and gray color scheme, but I haven't decided yet where I want to implement them.

The first actual change I made was to the font, because the default font makes a website seem underdeveloped in my opinion. After googling some different options for CSS that are accepted by all major browsers, I came to the decision to implement the "Trebuchet MS" font. I found it to be nice and simple, there weren't a lot of stylistic accents on the end of the letters, which made it easy to read.

Simply adding the color and font to the CSS was not enough to give my website a professional feel. The header was thin and didn't touch the menu. To fix this, I added 30px of padding and changed the margin to 0px. This gave the header a bigger, bolder area of black and brought it in contact with the menu below. The menu was too thin, and the options were too close together. To fix this, I added 10px of padding and changed the display to "flex" and the justification to "space-around." This made the menu an appropriate size in proportion to the header and spaced out the options equally according to the size of the screen. When a user makes the screen smaller, the menu options adjust to the width and remain equally centered. Finally, for a little bit of extra functionality, I added a hover style to the menu. When a user hovers over a menu item, the background color changes to black, as opposed to the gray on the rest of the menu.

Most of my work was done during class time. Conveniently, I also have group meetings for American Marketing Association on Tuesday nights after class. These do not start until 9:30 and class ends at 8:30, so every week I spend my down time finishing up some details of my website. This is when I normally google different things to mess around with and see how they work. I used to do this all on GitHub, but I got impatient waiting for the changes to make their way through GitHub, and to my final site. I recently copied and pasted my code into JSFiddle.net because it is much faster to type something in and press "Run" to see what effect it has through some quick trial and error.

In the immediate future, I want to implement a sticky menu, that keeps my menu bar available at the top of the page no matter how far users scroll down. This will be extremely useful for anyone visiting my site, especially if I am able to execute the parallax style later on. I also need to do some trial and error configuring the formatting of the "h2" and "p" tags to see what looks best. I'm going to play with centering the h2 as opposed to having it left aligned. I also might make it slightly larger and apply one of the blue colors from the color palette.Finally, I am going to play with some different margins for the paragraph text to try to center it more and change some of the alignment coding to make it adjust to the width of the screen, similar to the menu bar.

The first change I made since my last Work-In-Progress submission, was to add a professional headshot of myself to the About Me section. I thought that section was a little boring to be the home page, so the headshot adds some content. I thought the circular shape would be more elegant than a sharp square image. I put a border on it in order to add some of the blue color to the dark gray and black overall scheme. Lastly, I edited the About Me content from third person to first person, because I felt like it made more sense under my headshot and I added some more content about myself. After the changes to the About Me section, I also added a photo to In Class Assignment 1 and edited the In Class Assignment 2 text content.

When I was done revising old text content, I decided to change the active link color to the same blue as my headshot border. I implemented the active link feature for both the main header and the sub-head. Then, I changed the hover color to a lighter blue which complements the color scheme overall. The hover color also overrides the active color, so that users can still tell when they hover over the active link.

My third change to the site was to add some personal information to my footer, including a working email link that opens up a blank email directed to my Pitt account. Users can fill in their own content and contact me directly with the click of a button.

After editing the footer, it made sense to me to edit the header. The only edit I made was to enable the h1 to link to the home page, since most users are accustomed to this feature from other websites.

Once I was satisfied with the look of the website, I decided to experiment with the functionality of it. I mentioned in previous assignments that I wanted to implement parallax scrolling, which requires JavaScript, so I decided to try and teach myself a little bit. The first step was to move every sub-section of my site to the main page. I simply copied and pasted the code sequentially below the About Me section.

The next step was to enable the links in the header to send the user downward, rather than to a different page. To do this, I tagged each subhead with the “name=” code and then changed the links in the header to link to the respective names. I would like to make this a smooth scroll in the future, using JavaScript, but for now it is done using html and tags.

After I got those links to scroll the user rather than navigate them, I had to make a Sticky NavBar so that the user still had access to it all the way at the bottom of the page. This was my first encounter with JavaScript. I looked up a few different videos and articles on how to implement this, but I had trouble connecting a JavaScript file to my main index.html file. Interestingly enough, I figured out how to insert the JavaScript code at the bottom of my html code and it worked the way I wanted it to.

The NavBar now sticks to the top, but when the user scrolls far enough for it to stick, there is a slight jump on the page because of the missing space where the NavBar used to be. I still need to figure out how to fix this, but I would like to do so in a JavaScript file, not at the bottom of my html. Further, the sticky NavBar covers up my h2 when the link scrolls the user down to it. I need to figure out how to make the links scroll just above the h2, while keeping the sticky NavBar in a good spot. This, again, will require JavaScript which I would like to spend some more time figuring out.

The final thing I want to figure out how to do is change through sub-menu items while staying on the same page. For instance, if a user clicks WIP2 on the main menu, and is sent down here, and then clicks Photoshop Revision, I want everything else to stay the same, but have the WIP2 section change from Written Reflection to Photoshop Revision. I believe this is possible through JavaScript, but again, it will take some self-teaching to figure that out.
